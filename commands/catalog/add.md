将函数/API 记录到项目的 api-catalog 中。通过 subagent 执行，不影响主上下文。

流程：
1. 若 $ARGUMENTS 不为空，直接将其作为目标函数信息
2. 若 $ARGUMENTS 为空，从当前对话上下文中提取最近讨论/编写的函数名和所在文件路径
3. 使用 Task tool 启动 general-purpose subagent，prompt 中包含：
   - 目标函数名和文件路径
   - 以下完整操作指令

subagent 指令：
```
在项目中查找并读取目标函数的源码。然后将其记录到 .claude/api-catalog/ 中：
1. 检查 .claude/api-catalog/_index.md 是否存在，不存在则创建目录和空索引（表格格式：| 分类 | 说明 | 数量 |）
2. 读取 _index.md 判断该函数应归入哪个已有分类，若无合适分类则创建新分类（英文小写 kebab-case 命名）
3. 将函数条目追加到对应分类文件末尾，格式为：
   ### `函数签名`
   **用途**：一句话说明
   **示例**：最简调用示例
   **注意**：特殊约束（可选，如"用这个代替原生 Date()"）
   **位置**：文件路径:行号
4. 更新 _index.md 中对应分类的函数计数和最后更新日期
5. 返回记录结果：记录了什么函数、归入了哪个分类
```

4. 将 subagent 返回的结果展示给用户
